# 第 10 章：高级推理管道

> DistilledPipeline、ICLoraPipeline、KeyframeInterpolationPipeline、LipDubPipeline 等

## 10.1 DistilledPipeline（最快推理）

使用蒸馏模型，固定 8 步 sigma 时间表：

- **Stage 1**：8 步（`DISTILLED_SIGMA_VALUES` 9 个值）
- **Stage 2**：3 步（`STAGE_2_DISTILLED_SIGMA_VALUES` 4 个值）
- 无 `num_inference_steps` 参数（步数固定）
- 速度最快，质量略低于完整模型
- 需要蒸馏 checkpoint（`ltx-2.3-22b-distilled-1.1.safetensors`）

## 10.2 ICLoraPipeline（视频到视频变换）

使用 IC-LoRA（In-Context LoRA）进行风格/结构迁移：

- 输入：参考视频 + 目标条件
- 使用蒸馏模型
- `VideoConditionByReferenceLatent` 用于参考视频编码
- `reference_downscale_factor` 从 LoRA 元数据读取
- 支持注意力掩码降采样
- `skip_stage_2` 选项
- Stage 2 无 LoRA，使用 `combined_image_conditionings`（无 IC-LoRA 条件控制）

### 支持的 IC-LoRA

| LoRA | 功能 |
|------|------|
| Union Control | 通用图像条件（从参考图像/视频提取结构） |
| Motion Track | 运动轨迹控制 |
| Detailer | 细节增强 |
| Pose Control | 姿态控制 |
| Camera Control 系列 | 7 种镜头运动（推拉摇移等） |
| HDR | HDR 输出 |
| LipDub | 唇形同步 |

## 10.3 KeyframeInterpolationPipeline（关键帧插值）

在两幅或多幅关键帧图像之间生成平滑过渡：

- 两阶段管道（完整模型 + 蒸馏 LoRA）
- 两个阶段都使用 `image_conditionings_by_adding_guiding_latent`
- 所有帧都作为关键帧引导（非替换模式）
- 与 TI2Vid 的区别：TI2Vid 用 `combined_image_conditionings`（frame_idx=0 替换，其他引导）

## 10.4 A2VidPipelineTwoStage（音频到视频）

以音频文件为条件生成视频：

- 两阶段管道
- 两个阶段音频都冻结（`frozen=True, noise_scale=0.0`）
- 返回原始音频（不经 VAE 解码），无需 `AudioDecoder`

## 10.5 RetakePipeline（视频区域再生成）

对现有视频的特定时间区间进行再生成：

- 单阶段管道（完整或蒸馏模型均可）
- `TemporalRegionMask` 选择时间窗口
- `regenerate_video` / `regenerate_audio` 标志
- 可用于修复局部瑕疵

## 10.6 LipDubPipeline（唇形同步配音）

- 两阶段蒸馏管道
- IC-LoRA 应用于两个阶段
- 音频参考条件附加到 frozen audio-reference tokens
- 单 IC-LoRA，两个阶段均完整 IC-LoRA 视频条件控制
- Stage 2 音频冻结，使用 Stage 1 latent 作为初始状态

## 10.7 HDRICLoraPipeline（HDR 输出）

- 视频到视频的 HDR 输出
- 线性浮点帧通过 LogC3 逆解码输出
- 适合 EXR 导出和色调映射
