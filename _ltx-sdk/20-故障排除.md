# 第 20 章：故障排除

> OOM 修复、NaN 损失、配置错误、调试工具、最佳实践

## 20.1 OOM (Out of Memory)

| 症状 | 原因 | 解决方案 |
|------|------|----------|
| `CUDA out of memory` | VRAM 不足 | 降低 batch_size, 启用梯度检查点, 使用 8-bit text encoder |
| 训练中显存逐渐增长 | 内存泄漏 | 检查 dataloader, 确保 tensor 正确释放 |

### 7 种降低 VRAM 的方法

1. **降低 batch size**：`optimization.batch_size: 1`
2. **增加梯度累积**：`optimization.gradient_accumulation_steps: 8`
3. **启用梯度检查点**：`optimization.enable_gradient_checkpointing: true`
4. **8-bit 文本编码器**：`acceleration.use_8bit_text_encoder: true`
5. **更低的 LoRA 秩**：`model.lora.rank: 16`
6. **减少 LoRA 目标模块**：`model.lora.target_modules: [qkv]`
7. **FP8 量化**：`acceleration.quantization: fp8-cast`

## 20.2 NaN 损失

| 症状 | 原因 | 解决方案 |
|------|------|----------|
| Loss 变为 NaN | 学习率过高 | 降低 learning_rate (1e-5 ~ 5e-5) |
| Loss 在几步后爆炸 | 梯度爆炸 | 添加梯度裁剪, 检查数据范围 |
| 验证输出全黑/全白 | 数据预处理错误 | 检查 latent 值范围, 重建并目视检查 |

## 20.3 配置错误

| 错误 | 解决方案 |
|------|----------|
| `unknown field` | 检查配置键名拼写, 配置使用 `extra="forbid"` |
| 帧数不合法 | 确保 `frames % 8 == 1` (有效值: 1, 9, 17, 25...) |
| 分辨率不合法 | 确保宽高可被 32 整除 |
| 路径不存在 | 检查 model_path 和 text_encoder_path 是否为本地路径 |

## 20.4 训练问题

| 症状 | 诊断 | 解决方案 |
|------|------|----------|
| 训练速度慢 | GPU 利用率低 | 增加 batch size, 减少 dataloader workers |
| 质量不提升 | 学习率太低或数据不足 | 增加 lr 或数据量 |
| 恢复训练失败 | 检查点不兼容 | 确保使用相同的配置和 checkpoint |

## 20.5 调试工具

### GPU 监控

```bash
# 实时 GPU 状态
watch -n 1 nvidia-smi

# 按进程查看显存
nvidia-smi --query-compute-apps=pid,used_memory --format=csv
```

### 数据验证

```bash
# 解码并检查潜在质量
python scripts/decode_latents.py \
    --latents_dir precomputed/video_latents/ \
    --output_dir check_latents/ \
    --model_path checkpoint.safetensors
```

### 配置验证

```python
# Python 中验证配置
from ltx_trainer.config import LtxTrainerConfig
config = LtxTrainerConfig.from_yaml("configs/t2v_lora.yaml")
print(config.model_dump_json(indent=2))
```

## 20.6 最佳实践检查清单

- [ ] 先用小数据集 (5-10 个视频) 验证完整流程
- [ ] 检查验证输出，确保质量在提升
- [ ] 监控 GPU 利用率 (nvidia-smi)
- [ ] 定期备份检查点
- [ ] 使用 W&B 或 TensorBoard 记录实验
- [ ] 预处理后验证潜在质量 (decode_latents.py)
- [ ] 确保所有路径为本地路径
- [ ] 帧数满足 `frames % 8 == 1` 约束
- [ ] 分辨率可被 32 整除
