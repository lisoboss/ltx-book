# 第 18 章：自定义训练策略

> 策略模式实现、自定义条件与损失 API、注册与测试

## 18.1 何时需要自定义策略

大多数情况下，`FlexibleStrategy` 通过配置即可满足需求。仅在以下情况需要自定义策略：

- 全新的训练范式（如强化学习引导）
- 特殊的损失函数（非标准 velocity prediction）
- 复杂的多阶段训练流程
- 非标准的数据流或条件类型

判断标准：如果只是新的条件组合 → 使用 FlexibleStrategy 配置。如果需要修改损失函数或训练逻辑 → 创建自定义策略。

## 18.2 策略接口

```python
from ltx_trainer.training_strategies.base_strategy import (

class MyCustomStrategy(TrainingStrategy):

        """将批次数据转换为 ModelInputs (含 Modality 对象)"""
        ...

    def compute_loss(self, model, model_inputs: ModelInputs) -> torch.Tensor:
        """计算训练损失 (velocity prediction, MSE with masking)"""
        ...
```

## 18.3 配置类

```python
from typing import Literal
from pydantic import BaseModel

class MyCustomConfig(TrainingStrategyConfigBase):
    name: Literal["my_custom"] = "my_custom"
    custom_param: float = 1.0

    def get_data_sources(self) -> list[str]:
        return ["latents/video", "latents/custom"]
```

`get_data_sources()` 是单一数据源声明，既用于数据集连接也用于存在性验证。

## 18.4 注册策略

在 `src/ltx_trainer/training_strategies/__init__.py` 中完成三步注册：

```python
# 1. 导入
from .my_custom import MyCustomStrategy, MyCustomConfig

# 2. 添加到联合类型 (discriminated union)
TrainingStrategyConfig = Annotated[
    Union[FlexibleStrategyConfig, MyCustomConfig, ...],
    Field(discriminator="name"),
]

# 3. 添加到工厂函数
def get_training_strategy(config) -> TrainingStrategy:
    if isinstance(config, MyCustomConfig):
        return MyCustomStrategy(config)
    ...
```

## 18.5 测试自定义策略

```python
def test_my_custom_strategy():
    config = MyCustomConfig(
        video=ModalityConfig(is_generated=True, latents_dir="..."),
    )
    strategy = MyCustomStrategy(config)
    model_inputs = strategy.prepare_training_inputs(batch)
    loss = strategy.compute_loss(model, model_inputs)
    assert loss.requires_grad
```
