# 第 11 章：多模态引导

> CFG、STG、Modality CFG 的 API 与参数配置

## 11.1 引导机制概述

LTX-2 支持三种引导方式，可以在单个 Transformer 前向传播中组合使用。引导将多个去噪路径的预测结果融合，产生最终输出。

所有引导都通过 `GuidedDenoiser` 或 `FactoryGuidedDenoiser` 实现：

```python
from ltx_pipelines.utils.denoisers import GuidedDenoiser, FactoryGuidedDenoiser

# 静态引导器（HQ、A2Vid、Retake 非蒸馏）
denoiser = GuidedDenoiser(guider=guider)

# 每步动态创建（OneStage、TwoStages、Keyframe）
denoiser = FactoryGuidedDenoiser(guider_factory=factory)
```

## 11.2 CFG (Classifier-Free Guidance)

经典的稳定扩散引导机制：

```python
# 参数
cfg_scale = 3.0   # 视频默认
cfg_scale = 7.0   # 音频默认

# 效果
result = uncond_pred + cfg_scale * (cond_pred - uncond_pred)
```

- 更高的 scale → 更紧密跟随 prompt，但可能降低多样性
- 需要运行条件 + 非条件两次前向传播

## 11.3 STG (Spatio-Temporal Guidance)

扰动 Transformer 自注意力，增强时空一致性：

```python
stg_scale = 1.0          # 默认值
stg_blocks = [28]        # LTX-2.3 默认
stg_blocks = [29]        # LTX-2 默认
```

- 仅对指定块的自注意力注入扰动
- 扰动预测与原始预测混合
- HQ 管道禁用 STG（`stg_scale=0.0`）

## 11.4 Modality CFG

控制跨模态注意力的引导强度：

```python
modality_scale = 3.0     # 默认值
```

- 缩放视频↔音频交叉注意力的强度
- 在去噪过程中通过 `MultiModalGuider` 管理
- 影响音视频同步质量

## 11.5 组合引导与批次合并

引导去噪器将所有引导 pass **批量合并为单个 Transformer 调用**：

```
CFG only:       B=2 (cond + uncond)
CFG + STG:      B=3 (cond + uncond + perturbed)
CFG+STG+Modality: B=4 (所有组合)
全引导 pass:    最多 B=4

BatchedPerturbationConfig 控制每个样本跳过哪些注意力操作
结果被拆分并融合
```

## 11.6 SimpleDenoiser（无引导）

蒸馏管道和所有 Stage 2 使用 `SimpleDenoiser`：

```python
from ltx_pipelines.utils.denoisers import SimpleDenoiser

# 单次前向传播 (B=1)，无引导
denoiser = SimpleDenoiser()
result = denoiser(transformer, latent, sigma, context)
```

## 11.7 去噪结果类型

所有去噪器返回 `(video_result, audio_result)` 元组：

```python
from ltx_pipelines.utils.types import DenoisedLatentResult

# video_result.denoised   — 最终融合张量
# video_result.cond       — 条件预测（GuidedDenoiser）
# video_result.uncond     — 非条件预测（GuidedDenoiser）
# video_result.ptb        — 扰动预测（GuidedDenoiser）
# video_result.mod        — 模态引导预测（GuidedDenoiser）

# SimpleDenoiser 留下 cond/uncond/ptb/mod 为 None
```
