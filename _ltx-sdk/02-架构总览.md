# 第 2 章：架构总览

> 非对称双流 DiT 架构、跨模态注意力、数据流全景

## 2.1 高层设计

LTX-2 是一个**非对称双流 Diffusion Transformer**：

- **视频流**：14B 参数，处理 3D 视频数据（空间 + 时间）
- **音频流**：5B 参数，处理 1D 音频数据
- **总计**：~19B（LTX-2）/ ~22B（LTX-2.3）

两个流在 Transformer 块中通过**跨模态注意力**（Cross-Modality Attention）交互，使视频生成可以感知音频特征，反之亦然。

## 2.2 端到端数据流

```
输入文本 (Prompt)
      │
      ▼
┌─────────────────┐
│ Gemma 3 12B     │  文本编码 (3 块流水线)
│ 1. Tokenizer    │  ─→ 2. Feature Extractor (V1/V2)
│                 │  ─→ 3. Embeddings Processor (Connectors)
└────────┬────────┘
         │ video_embeds [4096-dim]  audio_embeds [2048-dim]
         ▼
┌─────────────────┐
│ Diffusion        │
│ Transformer      │  去噪循环
│ ┌─────────────┐  │
│ │ Self-Attn   │  │  文本交叉注意力 (所有块)
│ │ Text X-Attn │  │  音视频交叉注意力 (部分块)
│ │ A-V  X-Attn │  │  FFN
│ └─────────────┘  │
└────────┬────────┘
         │ video_latent [128ch]  audio_latent [8ch]
         ▼
┌─────────────────┬──────────────────┐
│ Video VAE       │ Audio VAE        │
│ Decoder         │ + Vocoder        │
│ (32x 压缩)      │ (HiFi-GAN/BigVGAN)│
└────────┬────────┴────────┬─────────┘
         │                 │
         ▼                 ▼
    输出视频            输出音频
```

## 2.3 潜在空间常量

这些值在整个 SDK 中统一使用：

| 常量 | 值 | 说明 |
|------|-----|------|
| Video latent channels | 128 | VAE 编解码器通道数 |
| Spatial compression | 32× (H, W) | 空间压缩比 |
| Temporal compression | 8× | 时间压缩比 |
| Frame constraint | `frames % 8 == 1` | 有效帧数：1, 9, 17, 25, 33... |
| Resolution constraint | W, H 可被 32 整除 | — |
| Audio latent channels | 8 | 音频潜在通道 |
| Audio mel bins | 16 | 梅尔频谱 bin 数 |
| Patchified token dim (video) | 128 | `128 × 1 × 1 × 1` |
| Patchified token dim (audio) | 128 | `8 × 16` |

## 2.4 版本差异：LTX-2 vs LTX-2.3

版本检测由 ltx-core 自动完成，通过 checkpoint config 识别：

| 组件 | LTX-2 (19B) | LTX-2.3 (22B) |
|------|-------------|---------------|
| Feature Extractor | `FeatureExtractorV1`：单一的 `aggregate_embed`，视频和音频输出相同 | `FeatureExtractorV2`：分离的 `video_aggregate_embed` + `audio_aggregate_embed`，逐 token RMSNorm |
| Caption Projection | 在 Transformer 内部 (`caption_projection`) | 在 Feature Extractor 内部（connector 之前） |
| Embeddings Connectors | 视频和音频同维度 | 分离维度 (`AudioEmbeddings1DConnectorConfigurator`) |
| Prompt AdaLN | 不存在 | 激活 — 用 `sigma` 调制对文本的交叉注意力 |
| Vocoder | HiFi-GAN (`Vocoder`) | BigVGAN v2 + 带宽扩展 (`VocoderWithBWE`) |

## 2.5 关键设计原则

1. **模态统一**：同一模型处理视频和音频，共享 Transformer 骨干
2. **条件灵活**：通过 `ConditioningItem` 系统统一表达所有条件类型
3. **版本自动检测**：代码路径由 checkpoint config 决定，无需手动指定版本
4. **内存友好**：`BlockStreamingWrapper` 支持逐块 GPU 计算，`BatchSplitAdapter` 支持批次拆分
