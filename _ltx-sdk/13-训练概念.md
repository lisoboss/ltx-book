# 第 13 章：训练概念

> 灵活策略、is_generated vs 冻结模态、LoRA vs 全量微调、版本兼容性

## 13.1 FlexibleStrategy（推荐）

`FlexibleStrategy` 是统一的训练框架，通过配置即可表达所有训练模式，无需为每个模式编写独立代码：

```yaml
# configs/t2v_lora.yaml 示例
training_strategy:
  name: flexible
  video:
    is_generated: true      # 视频将被生成（去噪目标）
    latents_dir: latents/video
    conditions: []           # 无条件 = 纯文本到视频
  audio:
    is_generated: false     # 音频不生成
```

### 核心概念

| 概念 | 说明 |
|------|------|
| `is_generated: true` | 该模态是训练目标（会被加噪并去噪） |
| `is_generated: false` | 该模态是条件（冻结，作为输入条件） |
| `conditions` | 每个模态的条件列表（first_frame, prefix, suffix, mask, reference 等） |

## 13.2 版本兼容性

训练器自动检测 checkpoint 版本（LTX-2 vs LTX-2.3），无需手动指定：

- 所有条件场景通过统一的 `FlexibleStrategy` 配置表达
- ltx-core 读取 checkpoint config 自动选择正确的架构组件
- 训练器无需版本特定的代码路径

## 13.3 LoRA vs 全量微调

### LoRA 训练（推荐）

- **参数量**：仅训练少量低秩适配器（~1-5% 参数量）
- **显存**：32GB 最低，80GB 推荐
- **速度**：较快，数据需求少
- **适用**：风格迁移、角色定制、新条件模式

```yaml
model:
  training_mode: lora
  lora:
    rank: 32
    alpha: 16
    target_modules:
      - qkv          # 注意力投影
      - ff           # 前馈网络（可选）
    dropout: 0.0
```

### 全量微调

- **参数量**：所有 22B 参数
- **显存**：80GB+，通常需要多 GPU
- **速度**：慢，数据需求大
- **适用**：全新能力、大规模域适应

### LoRA 目标模块选择

| 训练模式 | 推荐模块 | 说明 |
|----------|----------|------|
| T2V / I2V | qkv + ff (video) | 视频流全模块 |
| A2V | ff (cross-modal) | 跨模态注意力 |
| V2A / T2A | qkv + ff (audio) | 音频流全模块 |
| IC-LoRA | qkv (video) + ff (cross-modal) | 视频自注意力 + 跨模态 |

## 13.4 训练流程概览

```
1. 配置加载 → Pydantic 验证
2. 模型加载 → text encoder (Gemma), transformer, VAE
3. 文本嵌入缓存 → 预计算并卸载 Gemma LLM
4. 训练循环:
   ├── Embeddings connectors 应用
   ├── 策略准备 ModelInputs
   ├── Transformer 前向传播
   └── 策略计算 Loss (velocity prediction + MSE + masking)
5. 验证采样 → 条件生成并保存
6. 检查点保存 → safetensors + 配置
```
