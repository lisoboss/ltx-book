# 第 12 章：推理优化

> FP8 量化、torch.compile、梯度估计、内存管理

## 12.1 FP8 量化

在保持质量的同时大幅降低 VRAM 占用：

### FP8 Cast

```bash
# CLI
python -m ltx_pipelines.ti2vid_two_stages ... --quantization fp8-cast
```

```python
# Python
pipeline = TI2VidTwoStagesPipeline(
    ...,
    quantization=QuantizationPolicy.fp8_cast(),
)
```

- 将 bf16 checkpoint 即时降级为 FP8
- 内存占用降低约 2x
- 几乎无质量损失

### FP8 Scaled MM

```bash
--quantization fp8-scaled-mm
```

- 需要 FP8 checkpoint + Hopper GPU + TensorRT-LLM
- 使用硬件加速的缩放矩阵乘法
- 比 FP8 Cast 更快

## 12.2 torch.compile

在支持 Blackwell GPU（B200）上使用 `torch.compile` 加速：

```python
compile_config = {
    "backend": "inductor",
    "mode": "reduce-overhead",
    "fullgraph": True,
    "options": {
        "max_autotune": True,
        "epilogue_fusion": True,
        "coordinate_descent_tuning": True,
    },
}
```

## 12.3 梯度估计 (Gradient Estimation)

减少推理步数同时保持质量：

- 用更少的步数（20-30 而非 40）通过梯度估计补偿质量损失
- 在去噪循环中应用额外优化
- 参考 pipeline README 的 denoising loop optimization 部分

## 12.4 内存管理

### 跳过内存清理

如果 VRAM 充足，禁用阶段间的自动内存清理：

```python
pipeline = TI2VidTwoStagesPipeline(
    ...,
    skip_memory_cleanup=True,
)
```

### 块流式处理

自动将 Transformer 块从 CPU 流式传输到 GPU：

- **RAM 流式处理**：块存储在 pinned CPU RAM
- **磁盘流式处理**：块存储在磁盘（极端低内存场景）
- 每次 GPU 上只缓冲 2 个块

### 批次拆分

```python
# 当 max_batch_size=1 且需要 B=4（全引导）
# 自动拆分为 4 个串行 B=1 pass
# 减少层流式传输的 PCIe 传输开销
```

## 12.5 优化建议总结

| 场景 | 推荐方案 |
|------|----------|
| 快速原型 | DistilledPipeline（8 步） |
| 最佳质量 | TI2VidTwoStagesHQPipeline（Res2s, 15 步） |
| 低 VRAM | FP8 Cast + 块流式处理 |
| 平衡 | TI2VidTwoStagesPipeline + FP8 Cast + 梯度估计 |
| 单阶段 | TI2VidOneStagePipeline（无需高分辨率时） |
