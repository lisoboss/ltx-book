# 第 3 章：LTX-2 架构深度解析

> 理解你的创作工具，才能驾驭它。本章深入 LTX-2 的内部架构，帮助你理解每个参数背后的含义。

## 3.1 整体架构

LTX-2 是一个**非对称双流扩散 Transformer (Asymmetric Dual-Stream DiT)**，核心设计理念是：视频和音频虽然是紧密关联的两种模态，但具有不同的信息密度——视频需要更多参数处理空间+时间维度，音频则聚焦于时间维度。

```
                         文本 Prompt
                             │
                    ┌────────┴────────┐
                    │  Gemma 3 (12B)  │  文本理解
                    │ 多语言编码器      │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              │  Video Context│ Audio Context│
              │  [B,T,4096]   │  [B,T,2048]  │
              └──────┬────────┴──────┬───────┘
                     │               │
┌────────────────────┼───────────────┼──────────────────────┐
│              LTX-2 Transformer (48 Blocks)                 │
│                                                            │

│  │  3D RoPE (x,y,t)    │   │  1D RoPE (time)     │        │
```

## 3.2 核心组件深度解析

### 3.2.1 Transformer Block 执行顺序

每个双流 Block 执行 4 个操作，这 4 个操作的顺序和参数直接影响生成效果：

```
Block N 内部流程:
─────────────────────────────────────────────────────────────

Step 1: Self-Attention（自注意力）
  Video: RMSNorm → AdaLN(timestep) → Self-Attn (3D RoPE)
  Audio: RMSNorm → AdaLN(timestep) → Self-Attn (1D RoPE)
  → 各模态内部的空间/时间关系建模

Step 2: Text Cross-Attention（文本交叉注意力）
  Video: RMSNorm → Cross-Attn(Q=video, K/V=text_context[4096])
  Audio: RMSNorm → Cross-Attn(Q=audio, K/V=text_context[2048])
  → 文本语义注入。视频和音频收到不同的文本嵌入！

Step 3: Audio-Visual Cross-Attention（音视频交叉注意力）
  Video→Audio: Q=video, K/V=audio (1D temporal RoPE)
  Audio→Video: Q=audio, K/V=video (1D temporal RoPE)
  → 双向信息交换，实现音画同步
  → AdaLN 的 scale/shift 受对方模态的 hidden state 调制

Step 4: Feed-Forward Network（前馈网络）
  Video: RMSNorm → AdaLN → FFN (SwiGLU)
  Audio: RMSNorm → AdaLN → FFN (SwiGLU)
  → 特征精炼和非线性变换
```

### 3.2.2 RoPE 位置编码

这是理解 LTX-2 如何"知道"空间和时间的关键：

| 模态 | RoPE 类型 | 编码维度 | 含义 |
|------|-----------|----------|------|
| Video Self-Attn | **3D** (x, y, t) | dim/3 每轴 | 同时编码"在哪里"和"在什么时候" |
| Audio Self-Attn | **1D** (t) | dim | 仅编码时间顺序 |
| A↔V Cross-Attn | **1D** (t) | dim | 仅时间对齐，不做空间对齐 |

> **对 AI 动漫的启示**：3D RoPE 使模型天然理解"角色在画面左上角移动"这样的空间-时间关系。当你在 prompt 中描述位置和运动时，模型能准确对应。

### 3.2.3 AdaLN（自适应层归一化）

AdaLN 是扩散模型中常用的 conditioning 注入方式：

```
AdaLN(h, c) = scale(c) * LayerNorm(h) + shift(c)
```

其中 `c` 可以是：
- **Timestep embedding**：告诉模型当前处于去噪的哪个阶段（早期=结构，晚期=细节）
- **Cross-modality hidden state**：在 A↔V Cross-Attn 中，用对方模态的状态调制己方

> **对 AI 动漫的启示**：当你使用高 CFG scale 时，实际上是在强力放大文本条件对 AdaLN 的影响。过高的 CFG 会导致"过饱和"的动漫风格，适当降低可以保留更多自然细节。

### 3.2.4 Video VAE

```
Encoder: [B, 3, F, H, W] → [B, 128, F', H/32, W/32]
  where F' = 1 + (F-1)/8

示例:
  33 帧 512×512 → 128 通道 × 5 帧 × 16×16 latent
  49 帧 768×512 → 128 通道 × 7 帧 × 24×16 latent
  121 帧 512×512 → 128 通道 × 16 帧 × 16×16 latent
```

VAE 约束：
- 空间：W 和 H 必须是 32 的倍数
- 时间：F 必须满足 `(F-1) % 8 == 0`，即 F ∈ {1, 9, 17, 25, 33, 41, 49, 57, 65, 73, 81, 89, 97, 121, ...}

**Sequence Length 计算**（决定显存占用的核心指标）：

```
seq_len = (H/32) × (W/32) × ((F-1)/8 + 1)

Resolution      Frames    Seq_Len   估算显存
─────────────────────────────────────────────
512×512          49       16×16×7=1,792    轻量
768×512          49       24×16×7=2,688    中等
768×512          97       24×16×13=4,992   较高
1024×576         121      32×18×16=9,216   高
```

## 3.3 扩散过程与采样器

### 3.3.1 Sigma Schedule

LTX-2 使用的 LTX2Scheduler 根据 token 数量自适应调整 sigma schedule：

```python
from ltx_core.components.schedulers import LTX2Scheduler

scheduler = LTX2Scheduler()
sigmas = scheduler.execute(steps=40, latent=video_latent)
# sigmas 长度 = steps+1（含 sigma_max 到 sigma_min）
```

对于蒸馏模型（DistilledPipeline），使用固定的 8 步 sigma：

```python
# 蒸馏模型固定 schedule
DISTILLED_SIGMAS = [14.614, 6.475, 3.862, 2.328, 1.267, 0.589, 0.205, 0.042, 0.002]
# Stage 2（超分）
STAGE_2_DISTILLED_SIGMAS = [1.201, 0.546, 0.183, 0.030, 0.003]
```

### 3.3.2 采样器对比

| 采样器 | 步数 | 质量 | Pipeline |
|--------|------|------|----------|
| Euler | 40 (默认) | 高质量 | TI2VidTwoStagesPipeline |
| Res_2s (二阶) | 15 | 更高质量 | TI2VidTwoStagesHQPipeline |
| Distilled (8σ) | 8 | 良好 | DistilledPipeline, ICLoraPipeline |

> **对 AI 动漫的建议**：
> - 快速迭代/预览：用 Distilled (8 步)，几分钟出结果
> - 最终渲染：用 TI2VidTwoStagesHQPipeline (15 步 Res_2s)，质量最好
> - 平衡方案：用 TI2VidTwoStagesPipeline (40 步 Euler)，稳定可靠

## 3.4 Guidance 机制

### 3.4.1 三种 Guidance 的数学原理

```python
# CFG (Classifier-Free Guidance)
pred_final = pred_uncond + cfg_scale * (pred_cond - pred_uncond)

# STG (Spatio-Temporal Guidance)
# 扰动特定 attention block 后重新推理
pred_ptb = transformer(latent, perturbation=SKIP_VIDEO_SELF_ATTN)
pred_final = pred_ptb + stg_scale * (pred_cond - pred_ptb)

# Modality CFG（模态引导）
pred_final = pred_unsynced + modality_scale * (pred_cond - pred_unsynced)
```

### 3.4.2 动漫场景的推荐 Guidance 参数

```python
from ltx_core.components.guiders import MultiModalGuiderParams

# 动漫风格推荐参数
video_guider = MultiModalGuiderParams(
    cfg_scale=3.5,        # 比默认 3.0 稍高，增强动漫风格的 prompt 遵循
    stg_scale=1.0,        # 保持默认，稳定时间一致性
    rescale_scale=0.6,    # 比默认 0.7 稍低，让色彩更自然
    modality_scale=3.0,   # 如果生成音频则启用
    stg_blocks=[28],      # LTX-2.3 使用第 28 块（LTX-2 用 29）
)

# 如果需要更夸张的动漫风格
video_guider_strong = MultiModalGuiderParams(
    cfg_scale=5.0,        # 更高 = 更紧跟 prompt
    stg_scale=1.5,        # 更高 = 更强的结构保持
    rescale_scale=0.5,    # 更低 = 允许更风格化的色彩
    modality_scale=1.0,   # 纯视频不启用
    stg_blocks=[28],
)
```

## 3.5 Perturbation 系统

LTX-2 支持在推理时选择性跳过特定的注意力操作，这在 Guidance 和调试中非常有用：

```python
from ltx_core.guidance.perturbations import PerturbationType

perturbation_types = {
    "SKIP_VIDEO_SELF_ATTN":  "跳过视频自注意力",
    "SKIP_AUDIO_SELF_ATTN":  "跳过音频自注意力",
    "SKIP_A2V_CROSS_ATTN":   "跳过音频→视频交叉注意力",
    "SKIP_V2A_CROSS_ATTN":   "跳过视频→音频交叉注意力",
}
```

STG 机制正是通过 SKIP_VIDEO_SELF_ATTN perturbation 来实现的——对比正常推理和跳过视频自注意力的推理结果，二者之差作为"结构信号"引导生成。

## 3.6 模型加载流程

理解模型如何从磁盘加载到 GPU，有助于优化显存使用：

### 标准加载（SingleGPUModelBuilder）

```python
from ltx_core.loader import SingleGPUModelBuilder

builder = SingleGPUModelBuilder(
    model_class_configurator=model_configurator,
    model_path="models/ltx-2.3-22b-dev.safetensors",
).lora("loras/my_anime_style.safetensors", strength=0.8, sd_ops=lora_sd_ops)

model = builder.build(device=torch.device("cuda"))
# 此时完整模型 + LoRA 权重在 GPU 显存中
```

### Block Streaming 加载（省显存）

```python
from ltx_core.block_streaming import StreamingModelBuilder

builder = StreamingModelBuilder(
    model_class_configurator=model_configurator,
    model_path="models/ltx-2.3-22b-dev.safetensors",
    blocks_attr="transformer_blocks",
    blocks_prefix="transformer_blocks",
)

model = builder.build(
    device=torch.device("cuda"),
    dtype=torch.bfloat16,
    cpu_slots_count=4,   # CPU 只缓存 4 个 block
    gpu_slots_count=2,   # GPU 只驻留 2 个 block
)
```

> 48 个 Transformer Block + Block Streaming (gpu_slots=2)：显存占用降低约 96%。

## 3.7 量化策略

| 策略 | CLI 标志 | 显存节省 | 质量影响 | 硬件要求 |
|------|----------|----------|----------|----------|
| BF16 (无量化) | 默认 | 0% | 无 | 任何 CUDA GPU |
| FP8 Cast | `--quantization fp8-cast` | ~40% | 极小 | FP8 支持的 GPU |
| FP8 Scaled MM | `--quantization fp8-scaled-mm` | ~40% | 极小 | Hopper GPU + TensorRT-LLM |

```bash
# FP8 Cast — 通用方案
PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True \
python -m ltx_pipelines.ti2vid_two_stages \
    --quantization fp8-cast \
    --checkpoint-path models/ltx-2.3-22b-dev.safetensors \
    ...

# FP8 Scaled MM — Hopper GPU 最佳性能
PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True \
python -m ltx_pipelines.ti2vid_two_stages \
    --quantization fp8-scaled-mm \
    --checkpoint-path models/ltx-2.3-22b-dev.safetensors \
    ...
```
