# 第 5 章：图生视频：让动漫角色动起来

> 将静态动漫图像转化为动态视频，是 AI 动漫制作中最核心的能力。

## 5.1 图生视频原理

LTX-2 的图生视频（Image-to-Video, I2V）通过 **首帧条件（first_frame conditioning）** 实现：将输入图像编码为 latent，在去噪过程中，第一帧的 latent 被替换为图像 latent（timestep=0，无噪声），后续帧从噪声中逐步生成。

```
输入: [首帧图像] + [文本 Prompt]
              │
     ┌───────┴────────┐
     │ Video VAE      │
     │ Encoder        │
     │ 图像→Latent     │
     └───────┬────────┘
             │
     ┌───────┴────────────────────────────┐
     │ 初始化 Latent 序列                   │
     │ [Frame0=图像] [Frame1=噪声] ...      │
     │ [sigma=0]     [sigma=max]    ...     │
     └───────┬────────────────────────────┘
             │
     ┌───────┴────────┐
     │ 40步去噪       │
     │ Frame0 保持    │
     │ Frame1-N 演变  │
     └───────┬────────┘
             │
     ┌───────┴────────┐
     │ Video VAE      │
     │ Decoder        │
     │ Latent→像素     │
     └───────┬────────┘
             │
        输出视频
```

## 5.2 基础图生视频

### 使用 TI2VidTwoStagesPipeline

```python
from ltx_core.components.guiders import MultiModalGuiderParams
from ltx_core.loader import LTXV_LORA_COMFY_RENAMING_MAP, LoraPathStrengthAndSDOps
from ltx_core.model.video_vae import TilingConfig, get_video_chunks_number
from ltx_pipelines.ti2vid_two_stages import TI2VidTwoStagesPipeline
from ltx_pipelines.utils.args import ImageConditioningInput
from ltx_pipelines.utils.media_io import encode_video

# 蒸馏 LoRA 配置
distilled_lora = [
    LoraPathStrengthAndSDOps(
        "models/ltx-2.3-22b-distilled-lora-384-1.1.safetensors",
        0.6,
        LTXV_LORA_COMFY_RENAMING_MAP,
    ),
]

# 创建 pipeline
pipeline = TI2VidTwoStagesPipeline(
    checkpoint_path="models/ltx-2.3-22b-dev.safetensors",
    distilled_lora=distilled_lora,
    spatial_upsampler_path="models/ltx-2.3-spatial-upscaler-x2-1.1.safetensors",
    gemma_root="models/gemma-3-12b",
    loras=[],  # 稍后添加自定义动漫 LoRA
)

# Guidance 参数
video_guider = MultiModalGuiderParams(
    cfg_scale=3.0,
    stg_scale=1.0,
    rescale_scale=0.7,
    modality_scale=1.0,  # 纯视频
    stg_blocks=[28],
)

audio_guider = MultiModalGuiderParams(
    cfg_scale=7.0,
    stg_scale=1.0,
    rescale_scale=0.7,
    modality_scale=1.0,
    stg_blocks=[28],
)

# 从图像生成视频
num_frames = 73  # 约 3 秒 @24fps
frame_rate = 24.0
tiling_config = TilingConfig.default()

video, audio = pipeline(
    prompt="The anime girl slowly turns around, her silver hair catching the wind, "
           "cherry blossom petals dancing around her as she looks over her shoulder "
           "with a gentle smile. Soft golden sunset light. Studio Ghibli inspired style.",
    negative_prompt="worst quality, low quality, blurry, distorted, jpeg artifacts, "
                    "deformed face, bad anatomy, extra limbs",
    seed=42,
    height=512,
    width=768,
    num_frames=num_frames,
    frame_rate=frame_rate,
    num_inference_steps=40,
    video_guider_params=video_guider,
    audio_guider_params=audio_guider,
    images=[
        ImageConditioningInput(
            "keyframes/anime_girl_standing.jpg",  # 图像路径
            frame_idx=0,     # 放在第 0 帧（首帧）
            strength=1.0,    # 完全替换（1.0 = 100% 图像）
            crf=33,          # 编码质量（越高越接近原图）
        )
    ],
    tiling_config=tiling_config,
)

# 保存视频
encode_video(
    video=video,
    fps=frame_rate,
    audio=audio,
    output_path="outputs/anime_girl_turn.mp4",
    video_chunks_number=get_video_chunks_number(num_frames, tiling_config),
)
```

## 5.3 多图像条件组合

LTX-2 支持在多个帧位置放置参考图像：

```python
# 多帧图像条件
images = [
    ImageConditioningInput("frame_00.jpg", frame_idx=0, strength=1.0, crf=33),
    ImageConditioningInput("frame_24.jpg", frame_idx=24, strength=1.0, crf=33),
    ImageConditioningInput("frame_48.jpg", frame_idx=48, strength=0.7, crf=33),
    # strength < 1.0 = 软引导（guiding），而非硬替换（replacing）
]
```

### frame_idx=0 的特殊行为

- `frame_idx=0` 且 `strength=1.0`：**硬替换**（Replacing Latent），首帧完全等于输入图像
- `frame_idx > 0`：**软引导**（Guiding Latent），将图像信息作为引导信号而非直接替换

> 这是由 `combined_image_conditionings()` 函数自动处理的。代码位于：
> `packages/ltx-pipelines/src/ltx_pipelines/utils/helpers.py`

## 5.6 动漫图像预处理指南

### 5.6.1 图像尺寸要求

输入图像会自动 resize 并 center-crop 到目标尺寸。建议：

```
推荐动漫图像规格:
- 宽高比: 16:9 (768×432) 或 4:3 (768×576) 或 1:1 (512×512)
- 分辨率: 至少 1024px 短边（越高越好）
- 格式: JPG/PNG，RGB 色彩空间

动漫特有建议:
- 线稿保持清晰锐利（避免过度压缩的 JPG）
- 色彩平涂（cel-shaded）效果优于渐变渲染
- 避免过度的后期光效（让 AI 自行生成动态光效）
```

### 5.6.2 适合作为首帧的图像

```
✅ 适合的:
- 角色站立/坐姿的静态画面
- 半身或全身镜头（给 AI 留运动空间）
- 有明确朝向和姿态的角色
- 背景空间充裕的画面（让 AI 有余地生成运动）

❌ 不适合的:
- 极端特写（没有运动空间）
- 角色处于动作中间帧（AI 不知道前后动作）
- 画面被文字/对话框大面积覆盖
- 过度模糊或失焦的图像
```

## 5.7 实战：动漫角色转身

这是一个完整的动漫角色转身效果示例：

```python
# 参数调优方案
configs = {
    "gentle_turn": {
        "prompt": "slowly turning around with a gentle expression, "
                  "hair gently flowing, soft breeze, natural movement",
        "cfg_scale": 2.5,  # 低 CFG = 更自然的运动
        "num_frames": 73,  # 73帧/24fps ≈ 3秒
        "seed": 42,
    },
    "dramatic_reveal": {
        "prompt": "dramatically spinning around, hair whipping through the air, "
                  "intense expression, wind surging, dynamic camera movement, "
                  "speed lines and impact frame effect",
        "cfg_scale": 4.0,  # 高 CFG = 更强风格化
        "num_frames": 49,  # 49帧/24fps ≈ 2秒（快节奏）
        "seed": 123,
    },
    "walk_cycle": {
        "prompt": "walking forward with a confident stride, arms swinging "
                  "naturally, hair bouncing with each step, camera tracking "
                  "alongside, steady movement",
        "cfg_scale": 3.0,
        "num_frames": 97,  # 97帧/24fps ≈ 4秒（完整步态循环）
        "seed": 256,
    },
}

for name, cfg in configs.items():
    video, _ = pipeline(
        prompt=cfg["prompt"],
        seed=cfg["seed"],
        num_frames=cfg["num_frames"],
        video_guider_params=MultiModalGuiderParams(
            cfg_scale=cfg["cfg_scale"],
            stg_scale=1.0,
            rescale_scale=0.7,
            modality_scale=1.0,
            stg_blocks=[28],
        ),
        images=[ImageConditioningInput("character.jpg", 0, 1.0, 33)],
        ...
    )
    encode_video(video, ..., output_path=f"outputs/{name}.mp4")
```

## 5.8 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 角色变形/扭曲 | CFG 过高 | 降低 `cfg_scale` 到 2.0-2.5 |
| 动作不自然 | 缺乏 prompt 引导 | 添加具体动作描述 |
| 首帧与后续帧风格不一致 | 图像风格与模型训练分布差异大 | 使用动漫 LoRA；调高 `crf` |
| 画面闪烁/抖动 | STG 过低或帧数不足 | 提高 `stg_scale`；增加帧数 |
| 角色"漂浮"（不落地） | Prompt 缺乏地面参考 | 添加 "feet firmly on ground" |
| 背景扭曲 | 注意力被前景主导 | 增加背景描述权重 |
