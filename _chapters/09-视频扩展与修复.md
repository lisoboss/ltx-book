# 第 9 章：视频扩展、修复与 LipDub 配音

> 生成一段视频只是开始。AI 动漫制作中，你常常需要"续写"视频、修复不满意的时间段、或给角色配上精准的唇形同步语音。LTX-2 提供了完整的视频编辑能力。

## 9.1 视频扩展（Video Extension）

### 9.1.1 原理

视频扩展利用 `prefix` condition，将已有视频的前 N 帧作为上下文，生成后续帧：

```
已有视频: [████████████████] (帧 0-48)
                 ↓
扩展生成: [████████████████|▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓] (帧 49-96)

已有视频的 latent 作为 prefix condition (sigma=0, 无噪声)
后续帧从纯噪声开始去噪
Transformer 通过 Self-Attention 利用前缀信息生成连贯后续
```

### 9.1.2 训练视频扩展 LoRA

```yaml
# video_extend_lora.yaml
training_strategy:
  name: "flexible"
  video:
    is_generated: true
    latents_dir: "latents"
    conditions:
      - type: prefix
        temporal_boundary: 8  # 8 latent 帧 = 64 像素帧作为上下文
        probability: 1.0
  audio:
    is_generated: true
    latents_dir: "audio_latents"
```

```bash
# 训练
uv run python scripts/train.py configs/video_extend_lora.yaml
```

### 9.1.3 推理使用

训练好扩展 LoRA 后，可以通过 TI2VidPipeline 使用（将 prefix 条件映射到首帧图像条件）：

```python
# 视频扩展的推理方式：将已有视频的最后一帧作为 I2V 的首帧
# 结合扩展 LoRA 使用

extend_lora = LoraPathStrengthAndSDOps(
    "loras/video_extend_anime.safetensors",
    strength=0.8,
    sd_ops=LTXV_LORA_COMFY_RENAMING_MAP,
)

# 从已有视频提取最后一帧
last_frame = extract_last_frame("outputs/scene_part1.mp4")

pipeline = TI2VidTwoStagesPipeline(
    ...
    loras=[extend_lora, character_lora],
)

video, _ = pipeline(
    prompt="MYANIMECHAR continues walking down the forest path, the trees "
           "gradually thinning as she approaches a sunlit clearing ahead",
    images=[ImageConditioningInput(last_frame, frame_idx=0, strength=1.0, crf=33)],
    num_frames=49,
    ...
)
```

## 9.2 Retake（时间区域修复）

### 9.2.1 使用场景

```
常见修复需求:
├── "第 2-3 秒的画面崩了，只修这一段"
├── "角色在中间抬手了，我想改成挥手"
├── "那段背景闪烁太厉害，重新生成"
├── "某个时间段的音频有问题"
└── "只改视频不改音频" 或 "只改音频不改视频"
```

### 9.2.2 RetakePipeline

```python
from ltx_pipelines.retake import RetakePipeline
from ltx_pipelines.utils.media_io import decode_video_by_frame, read_audio

# 加载待修复的视频
source_frames = decode_video_by_frame("outputs/scene_draft.mp4")
source_audio = read_audio("outputs/scene_draft.mp4")

pipeline = RetakePipeline(
    checkpoint_path="models/ltx-2.3-22b-dev.safetensors",
    gemma_root="models/gemma-3-12b",
    loras=[character_lora],
)

video, audio = pipeline(
    prompt="MYANIMECHAR waves her hand gently instead of raising it, "
           "soft smile, cherry blossoms falling around her",
    negative_prompt="worst quality, low quality, deformed",
    seed=123,  # 可以换 seed 获得不同的修复结果
    height=512,
    width=768,
    num_frames=len(source_frames),
    frame_rate=24.0,
    num_inference_steps=40,
    source_video=source_frames,
    source_audio=source_audio,
    start_time=2.0,    # 从第 2 秒开始修复
    end_time=3.5,      # 到第 3.5 秒结束
    regenerate_video=True,   # 修复视频
    regenerate_audio=False,  # 保留原音频
    video_guider_params=video_guider,
    audio_guider_params=audio_guider,
)

encode_video(video, 24.0, audio, "outputs/scene_fixed.mp4")
```

### 9.2.3 CLI 方式

```bash
python -m ltx_pipelines.retake \
    --checkpoint-path models/ltx-2.3-22b-dev.safetensors \
    --gemma-root models/gemma-3-12b \
    --lora loras/my_character.safetensors 0.8 \
    --prompt "..." \
    --video-path outputs/scene_draft.mp4 \
    --start-time 2.0 \
    --end-time 3.5 \
    --num-inference-steps 40 \
    --output-path outputs/scene_fixed.mp4
```

### 9.2.4 Retake 约束

```
源视频约束:
- 帧数必须满足 8k+1 格式 (如 97, 193, ...)
- 分辨率必须是 32 的倍数
- start_time 和 end_time 必须在视频时长内
```

## 9.3 视频修复（Inpainting）

### 9.3.1 使用场景

```
动漫 Inpainting 常见用途:
├── 去除画面中的水印/字幕
├── 修复角色局部变形（手/脸/服装）
├── 替换背景中的物体
├── 擦除不需要的角色
└── 修复压缩伪影
```

### 9.3.2 训练

```yaml
# video_inpainting_lora.yaml
training_strategy:
  name: "flexible"
  video:
    is_generated: true
    latents_dir: "latents"
    conditions:
      - type: mask
        mask_dir: "video_masks"
        probability: 1.0
```

Mask 格式：`[F, H, W]` 的 `.pt` tensor，值 >0.5 = 保留区域（conditioning），≤0.5 = 重绘区域。

## 9.4 LipDub（唇形同步配音）

### 9.4.1 LipDub 原理

LipDub 是 LTX-2 独有的能力——给定一段角色说话的视频和一段新的音频，让角色的嘴唇动作精确匹配新音频，同时保持角色的外貌特征不变。

```
┌─────────────────────────────────────────────────────┐
│                  LipDub Pipeline                      │
├─────────────────────────────────────────────────────┤
│                                                       │
│  [参考视频]               [新音频]                     │
│  (角色说话)               (替换的语音/语言)            │
│      │                       │                        │
│      ├─ VAE Encode           ├─ Audio VAE Encode      │
│      │  → Video Latent       │  → Audio Latent        │
│      │  → Audio Latent       │                        │
│      │                       │                        │
│      │        ┌──────────────┴─────────────┐         │
│      └────────┤  Token 拼接                │         │
│               │  [Target音频 | Ref视频+音频]│         │
│               └──────────────┬─────────────┘         │
│                              │                        │
│               ┌──────────────┴─────────────┐         │
│               │  IC-LoRA LipDub            │         │
│               │  + 蒸馏模型                │         │
│               │  (两阶段)                   │         │
│               └──────────────┬─────────────┘         │
│                              │                        │
│                       视频输出 (唇形匹配新音频)         │
│                                                       │
└─────────────────────────────────────────────────────┘
```

### 9.4.2 使用 LipDubPipeline

```python
from ltx_pipelines.lipdub import LipDubPipeline
from ltx_core.loader import LTXV_LORA_COMFY_RENAMING_MAP, LoraPathStrengthAndSDOps

lipdub_lora = LoraPathStrengthAndSDOps(
    "models/loras/ltx-2.3-22b-ic-lora-lipdub-0.9.safetensors",
    strength=0.9,
    sd_ops=LTXV_LORA_COMFY_RENAMING_MAP,
)

pipeline = LipDubPipeline(
    distilled_checkpoint_path="models/ltx-2.3-22b-distilled-1.1.safetensors",
    spatial_upsampler_path="models/ltx-2.3-spatial-upscaler-x2-1.1.safetensors",
    gemma_root="models/gemma-3-12b",
    loras=[lipdub_lora],
)

video, audio = pipeline(
    prompt="The character speaks naturally with precise lip movements matching "
           "the provided audio, maintaining the same facial features and style",
    negative_prompt="worst quality, low quality",
    seed=42,
    height=512, width=768,
    frame_rate=24.0,
    reference_video_path="footage/character_speaking.mp4",
    reference_audio_path="dubbing/new_dialogue_japanese.wav",
    reference_strength=0.9,
)

encode_video(video, 24.0, audio, "outputs/lipdub_result.mp4")
```

### 9.4.3 CLI 方式

```bash
python -m ltx_pipelines.lipdub \
    --checkpoint-path models/ltx-2.3-22b-distilled-1.1.safetensors \
    --spatial-upsampler-path models/ltx-2.3-spatial-upscaler-x2-1.1.safetensors \

    --seed 42 \
    --output-path outputs/lipdub_result.mp4
```

### 9.4.4 AI 动漫中的 LipDub 应用场景

```
1. 多语言配音
   同一段画面 → 配音成日语/中文/英语/韩语

2. 台词修改
   保持画面不变 → 替换对白内容

3. AI 语音合成 + LipDub
   TTS 生成语音 → LipDub 匹配口型 → 完整配音

4. 虚拟主播/角色直播
   实时或离线 AI 语音 → LipDub 生成匹配的动漫口型
```

## 9.5 实战：动漫短片的后期修复工作流

```python
# 完整的后期修复流程

def post_production_workflow(scene_path, fixes):
    """
    scene_path: 场景视频路径
    fixes: 修复列表 [{"start": 2.0, "end": 3.5, "prompt": "..."}]
    """
    current_video = scene_path

    for i, fix in enumerate(fixes):
        print(f"Applying fix {i+1}/{len(fixes)}: {fix['start']}s-{fix['end']}s")

        source_frames = decode_video_by_frame(current_video)

        pipeline = RetakePipeline(
            checkpoint_path="models/ltx-2.3-22b-dev.safetensors",
            gemma_root="models/gemma-3-12b",
            loras=[character_lora],
        )

        video, audio = pipeline(
            prompt=fix["prompt"],
            negative_prompt="worst quality, low quality, deformed, blurry",
            seed=fix.get("seed", 42),
            height=fix.get("height", 512),
            width=fix.get("width", 768),
            num_frames=len(source_frames),
            frame_rate=24.0,
            num_inference_steps=40,
            source_video=source_frames,
            source_audio=None,  # 保留原音频
            start_time=fix["start"],
            end_time=fix["end"],
            regenerate_video=True,
            regenerate_audio=False,
            video_guider_params=video_guider,
            audio_guider_params=audio_guider,
        )

        current_video = f"outputs/fix_{i+1}.mp4"
        encode_video(video, 24.0, audio, current_video)

    return current_video

# 使用
fixes = [
    {"start": 1.5, "end": 2.5,
     "prompt": "MYANIMECHAR turns around gracefully, hair flowing, "
               "cherry blossoms swirling, warm sunset backlight"},
    {"start": 4.0, "end": 5.0,
     "prompt": "MYANIMECHAR smiles softly, eyes closing gently, "
               "a single tear rolling down, emotional moment"},
]

final_video = post_production_workflow("outputs/scene_raw.mp4", fixes)
```

## 9.6 视频修复约束速查

| 操作 | 帧数约束 | 分辨率约束 | 其他 |
|------|----------|-----------|------|
| Retake | 8k+1 | 32的倍数 | 源视频需预先 VAE 编码 |
| Inpainting | 8k+1 | 32的倍数 | Mask 需与 latent 尺寸匹配 |
| LipDub | 自动从参考视频获取 | 32的倍数 | 需要蒸馏模型 |
| Extension | 8k+1 | 32的倍数 | 需要扩展 LoRA |
