## 12.2 分辨率速查表

### VAE 约束

```
帧数必须满足: (F-1) % 8 == 0
即: F = 1, 9, 17, 25, 33, 41, 49, 57, 65, 73, 81, 89, 97, 105, 113, 121, 129, ...

空间分辨率必须满足: W % 32 == 0, H % 32 == 0

序列长度计算:
seq_len = (H/32) × (W/32) × ((F-1)/8 + 1)
```

### 常用分辨率与帧数组合

```
动漫推荐组合:

横屏 (16:9):
  768×432 × 49   = 24×13.5×7 ≈ 2,268 tokens  (轻量, 2秒)
  768×432 × 73   = 24×13.5×10 ≈ 3,240 tokens  (3秒)
  768×432 × 97   = 24×13.5×13 ≈ 4,212 tokens  (4秒)
  1024×576 × 49  = 32×18×7 ≈ 4,032 tokens     (2秒高清)

竖屏 (9:16) — 适合手机/短视频:
  432×768 × 49   = 13.5×24×7 ≈ 2,268 tokens
  432×768 × 73   = 13.5×24×10 ≈ 3,240 tokens

方形 (1:1) — 适合头像/封面:
  512×512 × 49   = 16×16×7 ≈ 1,792 tokens     (最轻量)
  512×512 × 97   = 16×16×13 ≈ 3,328 tokens

宽银幕 (2.35:1) — 电影感:
  1024×436 × 73  = 32×13.6×10 ≈ 4,352 tokens
```

## 12.3 常见问题 FAQ

### 推理 FAQ

**Q: 显存不足 (OOM) 怎么办？**

```
方案优先级:
1. 使用 --quantization fp8-cast (省 ~40% 显存)
2. 减小分辨率 (512×512 最稳定)
3. 减少帧数
4. 使用蒸馏模型 (DistilledPipeline)
5. 启用 Block Streaming (offload_mode)
6. 设置 PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True
```

**Q: 画面闪烁/抖动？**

```
- 提高 stg_scale (1.2-1.5)
- 增加帧数 (更多帧 = 更稳定)
- 降低 cfg_scale (减少过饱和)
- 使用固定 seed
- 检查输入图像是否一致
```

**Q: 角色变形/扭曲？**

```
- 降低 cfg_scale (2.0-3.0)
- 使用角色 LoRA
- 提高 crf (接近原图)
- 检查 prompt 是否矛盾
- 尝试不同的 seed
```

**Q: 风格不一致？**

```
- 使用 IC-LoRA 统一风格
- 训练风格 LoRA
- 在 prompt 中明确指定风格关键词
- 所有镜头使用相同的 style LoRA
```

**Q: 生成太慢？**

```
- 使用蒸馏模型 (5x 加速)
- 减少推理步数 (配合 gradient estimation)
- 启用 torch.compile
- 使用 HQ pipeline (15步 > 40步质量)
- 跳过 stage 间显存清理 (如果显存够)
```

### 训练 FAQ

**Q: 训练需要多少数据？**

```
最少: 30 个视频片段 (每个 5-10 秒)
推荐: 100 个片段
理想: 200+ 个片段，覆盖多角度/光照/动作

质量 > 数量: 50 个高质量片段 > 200 个低质量片段
```

**Q: LoRA rank 选多大？**

```
8-16: 轻量风格 LoRA
32: 标准角色 LoRA (推荐起步)
64: 复杂角色 + 服装 LoRA
128: 极高细节 (需要大量数据)
```

**Q: 训练 loss 不下降？**

```
- 降低 learning rate
- 检查数据质量 (crop 是否正确？caption 是否准确？)
- 检查预处理输出 (解码视频看看预处理是否正确)
- 增加 warmup_steps
```

**Q: 训练后角色不一致？**

```
- 检查训练数据中是否混入了其他角色
- 确保所有 caption 准确描述了角色特征
- 提高 LoRA rank
- 增加训练步数
- 确保推理时使用了正确的 trigger 词
```

## 12.5 Pipeline 功能矩阵

```
                     I2V  KF-Int IC-LoRA Audio Retake LipDub T2A
TI2VidOneStage       ✅    -      -      -     -      -      -
TI2VidTwoStages      ✅    -      -      -     -      -      -
TI2VidTwoStagesHQ    ✅    -      -      -     -      -      -
DistilledPipeline    ✅    -      -      -     -      -      -
KeyframeInterp       ✅    ✅     -      -     -      -      -
ICLoraPipeline       ✅    -      ✅     -     -      -      -
A2VidTwoStage        ✅    -      -      ✅    -      -      -
RetakePipeline       -     -      -      -     ✅     -      -
HDRICLoraPipeline    -     -      ✅     -     -      -      -
LipDubPipeline       -     -      ✅     ✅    -      ✅     -
T2AOneStage          -     -      -      ✅    -      -      ✅

✅ = 支持, - = 不支持

注意:
- I2V = 图生视频 (Image Conditioning)
- KF-Int = 关键帧插值 (Guiding Latents)
- IC-LoRA = 上下文 LoRA (视频参考条件)
- Audio = 音频驱动/生成
- Retake = 时间区域修复
- LipDub = 唇形同步配音
- T2A = 文本生成音频
```
