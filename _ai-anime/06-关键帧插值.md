# 第 6 章：关键帧插值：流畅的动作生成

> 在两张或多张动漫关键帧之间生成平滑过渡，是传统动画中最耗时的"中间帧"工作。LTX-2 可以将这个过程的效率提升百倍。

## 6.1 关键帧插值 vs 图生视频

| 特性 | 图生视频 (I2V) | 关键帧插值 |
|------|---------------|-----------|
| 输入 | 1 张首帧 + prompt | 2+ 张关键帧 + prompt |
| 控制力 | 仅控制起点 | 同时控制起点和终点 |
| 适用场景 | 自由动作生成 | 精确控制的关键动作过渡 |
| 帧位置 | 第 0 帧 = 图像 | 任意帧可以有图像 |
| 实现方式 | 替换首帧 latent | 全部帧使用 guiding latent |

## 6.2 关键帧插值原理

```
输入: [Frame0 关键帧] [Frame24 关键帧] [Frame48 关键帧]
                     │
          ┌──────────┴──────────┐
          │  Video VAE Encoder  │  ← 编码所有关键帧
          └──────────┬──────────┘
                     │
          ┌──────────┴──────────────────────────────┐
          │  初始化 Latent 序列                       │
          │  Frame0:  guiding=图象latent, sigma=max   │
          │  Frame1-23: 纯噪声                        │
          │  Frame24: guiding=图象latent, sigma=max   │
          │  Frame25-47: 纯噪声                       │
          │  Frame48: guiding=图象latent, sigma=max   │
          └──────────┬──────────────────────────────┘
                     │
          ┌──────────┴──────────┐
          │  40 步去噪           │
          │  所有帧同时演变       │
          │  关键帧位置受引导     │
          └──────────┬──────────┘
                     │
                输出视频 (平滑插值)
```

> **关键区别**：KeyframeInterpolationPipeline 使用 `image_conditionings_by_adding_guiding_latent()`，所有帧都作为 guiding signal（而非替换）。这确保了关键帧之间有平滑的过渡。

## 6.3 基础关键帧插值

### Python API

```python
from ltx_core.components.guiders import MultiModalGuiderParams
from ltx_core.loader import LTXV_LORA_COMFY_RENAMING_MAP, LoraPathStrengthAndSDOps
from ltx_core.model.video_vae import TilingConfig, get_video_chunks_number
from ltx_pipelines.keyframe_interpolation import KeyframeInterpolationPipeline
from ltx_pipelines.utils.args import ImageConditioningInput
from ltx_pipelines.utils.media_io import encode_video

distilled_lora = [
    LoraPathStrengthAndSDOps(
        "models/ltx-2.3-22b-distilled-lora-384-1.1.safetensors",
        0.6,
        LTXV_LORA_COMFY_RENAMING_MAP,
    ),
]

pipeline = KeyframeInterpolationPipeline(
    checkpoint_path="models/ltx-2.3-22b-dev.safetensors",
    distilled_lora=distilled_lora,
    spatial_upsampler_path="models/ltx-2.3-spatial-upscaler-x2-1.1.safetensors",
    gemma_root="models/gemma-3-12b",
    loras=[],
)

video_guider = MultiModalGuiderParams(
    cfg_scale=3.0,
    stg_scale=1.0,
    rescale_scale=0.7,
    modality_scale=1.0,
    stg_blocks=[28],
)

# 关键帧定义
keyframes = [
    ImageConditioningInput("keyframes/pose_01_start.jpg", frame_idx=0, strength=1.0, crf=33),
    ImageConditioningInput("keyframes/pose_02_middle.jpg", frame_idx=24, strength=1.0, crf=33),
    ImageConditioningInput("keyframes/pose_03_end.jpg", frame_idx=48, strength=1.0, crf=33),
]

video, audio = pipeline(
    prompt="An anime warrior drawing a katana in one fluid motion, the blade "
           "gleaming as it exits the sheath, determined expression, dynamic "
           "camera angle tilting upward, wind effect emphasizing the motion",
    negative_prompt="worst quality, low quality, blurry, deformed",
    seed=42,
    height=512,
    width=768,
    num_frames=49,
    frame_rate=24.0,
    num_inference_steps=40,
    video_guider_params=video_guider,
    audio_guider_params=MultiModalGuiderParams(
        cfg_scale=7.0, stg_scale=1.0, rescale_scale=0.7,
        modality_scale=1.0, stg_blocks=[28],
    ),
    images=keyframes,
    tiling_config=TilingConfig.default(),
)

encode_video(
    video=video, fps=24.0, audio=audio,
    output_path="outputs/katana_draw_interpolated.mp4",
    video_chunks_number=get_video_chunks_number(49, TilingConfig.default()),
)
```

### CLI 方式

```bash
python -m ltx_pipelines.keyframe_interpolation \

    --num-inference-steps 40 \
    --seed 42 \
    --output-path outputs/keyframe_interp.mp4
```

## 6.4 动漫关键帧设计策略

### 6.4.1 关键帧数量与间隔

```
动画类型建议:

┌──────────────────┬──────────────┬──────────────┬─────────────┐
│ 动画类型          │ 关键帧间隔    │ 49帧中的位置  │ 适用场景      │
├──────────────────┼──────────────┼──────────────┼─────────────┤
│ 慢动作/抒情       │ 24帧         │ 0, 24, 48    │ 情感特写      │
│ 正常动作          │ 12-16帧      │ 0,12,24,36,48│ 日常对话/行走 │
│ 快速动作          │ 6-8帧        │ 3-5帧间隔    │ 战斗/运动     │
│ 极限姿势→姿势     │ 仅首尾关键帧  │ 0, 48        │ 变身/觉醒     │
└──────────────────┴──────────────┴──────────────┴─────────────┘
```

### 6.4.2 动漫中的"预备-动作-缓冲"（Anticipation-Action-Follow-through）

这是动画 12 原则的核心：

```
时间线示例（49帧 @24fps ≈ 2秒）:

帧 0:   预备动作 (Anticipation) — 角色身体下蹲/后仰
帧 8:   动作起始 — 开始发力
帧 16:  动作中点 — 速度最快处
帧 24:  接触/击中 — 关键帧
帧 32:  缓冲开始 (Follow-through) — 惯性延续
帧 40:  缓冲继续 — 衣物/头发仍在运动
帧 48:  恢复静止或连接下一动作

关键帧: 0, 8, 24, 48
→ LTX-2 自动生成 1-7, 9-23, 25-47 的中间帧
```

### 6.4.3 生成动漫关键帧素材

关键帧图像可以通过以下方式获得：

```python
# 方案 1: 从漫画/动画中手动选择关键帧
# 用 Photoshop/Krita 等工具裁剪调整到统一尺寸

# 方案 2: 用 Stable Diffusion / Flux 等 T2I 模型生成关键帧
# 用相同的 seed 和 prompt，在不同 denoising 阶段停止

# 方案 3: 用 3D 软件 (Blender) 的卡通渲染输出关键帧
# 然后 AI 插值生成中间帧（这是日本动画公司常用的 AI 辅助方式）

# 方案 4: 手绘粗略关键帧 → AI 插值+细化
```

## 6.5 实战：动漫战斗场景插值

```python
# 完整战斗场景：拔刀 → 斩击 → 收刀

battle_keyframes = {
    "scene_1_draw": [
        # 预备: 手按刀柄
        ImageConditioningInput("battle/draw_start.jpg", frame_idx=0, strength=1.0, crf=33),
        # 拔刀中: 刀身半出
        ImageConditioningInput("battle/draw_mid.jpg", frame_idx=8, strength=1.0, crf=33),
        # 拔刀完成: 刀身全出，起步姿势
        ImageConditioningInput("battle/draw_end.jpg", frame_idx=16, strength=1.0, crf=33),
    ],
    "scene_2_slash": [
        # 斩击预备: 刀高举
        ImageConditioningInput("battle/slash_ready.jpg", frame_idx=0, strength=1.0, crf=33),
        # 斩击命中: 刀身砍入
        ImageConditioningInput("battle/slash_hit.jpg", frame_idx=8, strength=1.0, crf=33),
        # 斩后姿势: 刀身甩出
        ImageConditioningInput("battle/slash_follow.jpg", frame_idx=16, strength=1.0, crf=33),
    ],
    "scene_3_sheathe": [
        # 血振: 甩刀
        ImageConditioningInput("battle/blood_flick.jpg", frame_idx=0, strength=1.0, crf=33),
        # 纳刀: 刀入鞘一半
        ImageConditioningInput("battle/sheathe_mid.jpg", frame_idx=8, strength=1.0, crf=33),
        # 纳刀完成: 刀全入鞘，咔嚓声
        ImageConditioningInput("battle/sheathe_end.jpg", frame_idx=16, strength=1.0, crf=33),
    ],
}

prompts = {
    "scene_1_draw": "Anime samurai slowly drawing katana from sheath, the metal "
                    "gleaming as it emerges, dramatic tension building, camera "
                    "slowly dollying in, dark atmospheric lighting with rim light "
                    "on the blade, ufotable action style",
    "scene_2_slash": "Explosive slash attack, the blade cutting through the air "
                     "with incredible speed, blue energy trail following the arc, "
                     "impact frame effect at the moment of strike, speed lines, "
                     "dynamic camera shake, dramatic cel shading with bold outlines",
    "scene_3_sheathe": "Samurai performing chiburi (blood flick) and noto (sheathing), "
                       "fluid graceful movement, blood droplets scattering in slow "
                       "motion, soft moonlight, serene expression, the katana clicks "
                       "into the sheath with finality",
}

for scene_name, keyframes in battle_keyframes.items():
    video, _ = pipeline(
        prompt=prompts[scene_name],
        negative_prompt="worst quality, low quality, blurry",
        seed=42,
        height=512, width=768, num_frames=17,  # 紧凑的短片段
        frame_rate=24.0,
        num_inference_steps=40,
        video_guider_params=video_guider,
        audio_guider_params=audio_guider,
        images=keyframes,
        tiling_config=TilingConfig.default(),
    )
    encode_video(
        video=video, fps=24.0, audio=None,
        output_path=f"outputs/battle_{scene_name}.mp4",
        video_chunks_number=get_video_chunks_number(17, TilingConfig.default()),
    )
```

## 6.6 关键帧插值最佳实践

### 6.6.1 角色一致性

```
关键帧一致性的三个层次:

1. 服装/发型一致（基础）
   → 所有关键帧使用同一角色的图像

2. 比例/透视一致（中级）
   → 关键帧的角色大小和视角保持连贯

3. 光线/色彩一致（高级）
   → 所有关键帧使用相同的光源方向和色调
```

### 6.6.2 运动弧线

```
好的关键帧插值需要遵循运动弧线原则:

❌ 直线运动（不自然）:
   Frame0: 角色在左侧 → Frame24: 角色在右侧

✅ 弧线运动（自然）:
   Frame0: 角色在左侧 → Frame12: 角色在中间偏上 → Frame24: 角色在右侧
   （形成弧形运动轨迹）
```

### 6.6.3 速度变化（Slow In & Slow Out）

```
推荐关键帧密度:

匀速运动:        关键帧均匀分布 │···│···│···│···│
慢入慢出:        两端密中间疏   │·│··│····│··│·│
加速运动:        逐渐变疏       │·│··│···│····│
减速运动:        逐渐变密       │····│···│··│·│
```

## 6.7 与其他 Pipeline 的协作

关键帧插值可以与图生视频组合使用：

```
工作流: 关键帧生成 → 关键帧插值 → 视频修复

Step 1: 用 SD/Flux 生成角色在不同姿势的关键帧图像
Step 2: 用 TI2VidOneStage 将每张关键帧生成短片段（作为"种子视频"）
Step 3: 从各片段中提取首尾帧作为新的关键帧
Step 4: 用 KeyframeInterpolationPipeline 生成流畅过渡
Step 5: 用 RetakePipeline 修复不满意的时间段
```
